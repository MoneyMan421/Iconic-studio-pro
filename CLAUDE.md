# Iconic Studio Pro — Project Context Document

---

## Project Overview

**Iconic Studio Pro** is a premium Flutter/Dart icon editor application featuring real-time diamond refraction shaders (GLSL). Users can import images, apply shader-based visual effects (brightness, contrast, saturation, blur, diamond refraction parameters), and export the result. The app includes a freemium model (2 free imports, then paywall), user authentication, and Firebase backend integration.

A secondary utility repository (**AndroidStringsTranslate**) exists in this account — a standalone Python script for translating Android `strings.xml` entries to multiple languages via Google Translate. It is **not** part of the Flutter app.

## Architecture

### High-Level Pattern
- **Single-page editor architecture**: The main studio UI lives entirely in `lib/main.dart`. All editor concerns (state, rendering, export, paywall) are co-located there.
- **Immutable state with copyWith**: `EditorState` is a pure value object; all mutations produce a new instance via `copyWith()`.
- **Platform-conditional export**: `export_helper.dart` uses Dart conditional imports to route to `export_io.dart` (native) or `export_web.dart` (web).
- **Auth gate pattern**: `AuthGate` wraps all protected screens; it resolves auth state asynchronously before routing.

### Key Data Flow
```
User action (slider/button)
  → setState() updates EditorState
  → PreviewCanvas re-renders with new shader uniforms
  → Export: RepaintBoundary capture → PNG bytes → export_helper
```

### Auth Flow
```
App start → AuthGate loads SharedPreferences
  → isLoggedIn == false → AuthScreen (SignUp / Login)
  → isLoggedIn == true  → StudioPage
```
> ⚠️ Current auth is **local only** (SharedPreferences + password hash). Firebase Auth is included as a dependency but the migration from local auth to Firebase Auth may be incomplete.

### Shader Pipeline
- `diamond_master.frag` is a GLSL fragment shader registered in `pubspec.yaml` under `flutter.shaders`.
- `PreviewCanvas` uses `flutter_shaders`/`AnimatedSampler` to pass uniforms each frame.
- A `Ticker` drives `_elapsedSeconds` for animated time uniform.

Handles local persistence of `EditorState` (likely via `shared_preferences`). Exposes `load()` and `save()` methods called from `_StudioPageState`.

## Coding Conventions

### Dart/Flutter
- **Immutable state**: Always use `EditorState.copyWith()` — never mutate fields directly.
- **Colors**: Use `AppColors.*` constants exclusively; no raw `Color(0x...)` literals in UI code.
- **Export routing**: Always go through `export_helper.dart`, not platform-specific files.
- **`withValues(alpha:)`**: Used instead of deprecated `withOpacity()` — follow this pattern.
- **Widget decomposition**: Prefer small, focused widgets (`_StatItem`, `_AuthField`, `_GoldButton` as examples).
- **`const` constructors**: Use wherever possible.
- **State management**: Currently `setState` + `ChangeNotifier` — no external state management library (no Riverpod/Bloc/Provider).

### Tests
- **Tests must pump `MaterialApp(home: const StudioPage())`** directly — NOT `const IconStudioPro()` or `const AuthGate(...)`, because the async SharedPreferences load prevents the studio UI from rendering in a single `pumpWidget` call.
- Test file: `test/widget_test.dart`.

### Git / Branch Strategy
- Active development branches:
  - `main_queued` on `Iconic-studio-pro` — synced from `main`; AI agent PRs branch from here (e.g. `feat/fix-all-known-issues-*`)
  - `develop_queued` on `AndroidStringsTranslate` — synced from `develop`
- Feature branches use the pattern `feat/<description>-<short-hash>-queued`
- Never commit `android/key.properties`, `*.keystore`, or `*.jks` (in `.gitignore`).
- `lib/firebase_options.dart` contains real keys — treat as sensitive; generated by FlutterFire CLI.
- `android/app/google-services.json` must not be committed (downloaded from Firebase Console).

## Development Workflow

### First-Time Setup
```bash
# 1. Get dependencies
flutter pub get

# 2. Configure Firebase (generates lib/firebase_options.dart)
dart pub global activate flutterfire_cli
flutterfire configure --project=YOUR_FIREBASE_PROJECT_ID

# 3. Place google-services.json
# Download from: Firebase Console → Project Settings → Android → Download
cp ~/Downloads/google-services.json android/app/google-services.json

# 4. Run
flutter run
```

### Firebase Services to Enable
- Authentication → Email/Password
- Firestore Database → Production mode
- Storage → Production mode

See `FIREBASE_SETUP.md` for full Firestore and Storage security rules.

### CI Pipeline (`.github/workflows/ci.yml`)
```
flutter analyze → flutter test → build Android → build iOS → build web
```

### Adding a New Screen
1. Create `lib/your_screen.dart`
2. Use `AppColors.*` for all colors
3. Route from `AuthGate` child or `StudioPage` navigation
4. Add widget tests that pump `MaterialApp(home: const YourScreen())`

### Modifying the Shader
- Edit `shaders/diamond_master.frag`
- Shader is declared in `pubspec.yaml` under `flutter.shaders` — no additional registration needed
- Uniforms are passed from `PreviewCanvas` / `_PreviewCanvasState`; update both sides when adding new uniforms

### Modifying EditorState
1. Add field to `EditorState` with a default value
2. Add the parameter to `copyWith()`
3. Pass the new uniform to the shader in `PreviewCanvas` if shader-relevant
4. Update `EditorStorage.save()`/`load()` to persist the new field

## AndroidStringsTranslate (Secondary Repo)

**Completely unrelated to the Flutter app.** A standalone Python utility:

- **File**: `translate.py`
- **Purpose**: Reads Android `strings.xml` entries from a `source` file, translates them to multiple languages (nl, fr, de, pl, es) using `deep_translator` (Google Translate), and writes results to a timestamped `result_*.txt` file.
- **Usage**: `python translate.py`
- HTML tags are temporarily replaced with UUID tokens before translation to preserve XML structure, then restored afterward.
- Active branch: `develop_queued` (synced from `develop`)
